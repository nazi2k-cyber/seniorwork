# 핵심 모듈 분리 계획서

> 시니어 특화 하이퍼로컬 일자리 매칭 앱 — 모듈 아키텍처 설계

---

## 1. 개요

본 문서는 시니어 특화 하이퍼로컬 일자리 매칭 앱의 핵심 기능을 독립적인 모듈로 분리하여 설계하기 위한 계획서입니다. 각 모듈은 독립적으로 개발·배포·테스트가 가능하도록 설계하며, 모듈 간 결합도를 최소화하고 응집도를 극대화합니다.

### 분리 목적
- 기능별 독립 개발 및 배포 가능
- 팀 단위 병렬 개발 효율 극대화
- 장애 격리 및 유지보수 용이성 확보
- 추후 마이크로서비스 전환 대비

## 3. 핵심 모듈 상세 설계

### 3.1 인증/회원 모듈 (`auth`)

| 항목 | 내용 |
|:---|:---|
| **책임** | 회원가입, 로그인, 본인인증, 세션 관리 |
| **핵심 기능** | 간편 로그인(카카오/네이버), 휴대폰 본인인증, 보호자 대리 가입 |
| **외부 의존** | Firebase Auth, OAuth 2.0 Provider |
| **시니어 특화** | 최소 입력 단계(3단계 이내), 자동 로그인 유지, 큰 버튼 UI |

```
auth/
├── data/
│   ├── repository/          # AuthRepository 구현
│   └── datasource/          # Firebase, OAuth 데이터소스
├── domain/
│   ├── model/               # User, AuthToken
│   └── usecase/             # LoginUseCase, RegisterUseCase
└── presentation/
    ├── login/
    └── register/
```

### 3.2 프로필 관리 모듈 (`profile`)

| 항목 | 내용 |
|:---|:---|
| **책임** | 사용자 프로필 CRUD, 이력 관리 |
| **핵심 기능** | 간편 프로필 등록(이력서 대체), 프로필 사진, 보유 기술 태그 |
| **외부 의존** | Cloud Storage (이미지), auth 모듈 |
| **시니어 특화** | 음성 입력(STT)으로 프로필 작성, 최소한의 필수 항목 |

```
profile/
├── data/
│   ├── repository/
│   └── datasource/
├── domain/
│   ├── model/               # Profile, Skill
│   └── usecase/             # UpdateProfileUseCase
└── presentation/
    ├── edit/
    └── view/
```

### 3.3 일자리 모듈 (`job`)

| 항목 | 내용 |
|:---|:---|
| **책임** | 일자리 등록, 조회, 검색, 지원 |
| **핵심 기능** | 동네 기반 일자리 목록, 카테고리 필터, 간편 지원 |
| **외부 의존** | location 모듈, matching 모듈 |
| **시니어 특화** | 시니어 적합 일자리 큐레이션, 큰 카드형 리스트, 음성 검색 |

```
job/
├── data/
│   ├── repository/
│   └── datasource/
├── domain/
│   ├── model/               # Job, JobCategory, Application
│   └── usecase/             # SearchJobUseCase, ApplyJobUseCase
└── presentation/
    ├── list/
    ├── detail/
    └── register/            # 구인자용 일자리 등록
```

### 3.4 매칭 엔진 모듈 (`matching`)

| 항목 | 내용 |
|:---|:---|
| **책임** | 구직자-구인자 매칭 알고리즘 |
| **핵심 기능** | 거리 기반 매칭, 기술 적합도 스코어링, 역제안(구인자→구직자) 알림 |
| **외부 의존** | location 모듈, profile 모듈, job 모듈 |
| **시니어 특화** | 출퇴근 거리 가중치 상향, 업무 강도 필터, 시니어 선호 직종 우선 노출 |

```
matching/
├── data/
│   ├── repository/
│   └── datasource/
├── domain/
│   ├── model/               # MatchResult, MatchScore
│   └── usecase/             # CalculateMatchUseCase, SuggestJobUseCase
└── engine/
    ├── scorer/              # 매칭 점수 계산 엔진
    └── filter/              # 필터링 규칙
```

### 3.5 위치 기반 서비스 모듈 (`location`)

| 항목 | 내용 |
|:---|:---|
| **책임** | GPS 위치 관리, 지역 설정, 반경 검색 |
| **핵심 기능** | 현재 위치 자동 감지, 동네 설정(행정동 기반), Geo-query 반경 검색 |
| **외부 의존** | Google Maps/Kakao Map API, Firebase GeoFire |
| **시니어 특화** | 기본 반경 좁게 설정(도보 15분 이내), 지도 UI 큰 핀/마커 |

```
location/
├── data/
│   ├── repository/
│   └── datasource/          # GPS, Map API
├── domain/
│   ├── model/               # Location, Region, GeoPoint
│   └── usecase/             # GetNearbyUseCase, SetRegionUseCase
└── presentation/
    └── map/
```

### 3.6 실시간 채팅 모듈 (`chat`)

| 항목 | 내용 |
|:---|:---|
| **책임** | 구직자-구인자 1:1 실시간 채팅 |
| **핵심 기능** | 지원 즉시 채팅방 생성, 텍스트/이미지 전송, 읽음 확인 |
| **외부 의존** | Firebase Realtime DB / Firestore, notification 모듈 |
| **시니어 특화** | 큰 폰트 채팅 UI, 음성 메시지 지원, 자주 쓰는 문구 빠른 전송 |

```
chat/
├── data/
│   ├── repository/
│   └── datasource/          # Firebase Realtime DB
├── domain/
│   ├── model/               # ChatRoom, Message
│   └── usecase/             # SendMessageUseCase, LoadChatUseCase
└── presentation/
    ├── room_list/
    └── conversation/
```

### 3.7 리뷰/평가 모듈 (`review`)

| 항목 | 내용 |
|:---|:---|
| **책임** | 상호 리뷰, 매너 평가, 노쇼 방지 |
| **핵심 기능** | 별점 + 매너 태그 평가, 리뷰 조회, 매너 온도 산출 |
| **외부 의존** | auth 모듈, job 모듈 |
| **시니어 특화** | 탭 기반 간편 평가(별점 + 이모티콘), 텍스트 리뷰 선택적 |

```
review/
├── data/
│   ├── repository/
│   └── datasource/
├── domain/
│   ├── model/               # Review, MannerScore
│   └── usecase/             # SubmitReviewUseCase, GetScoreUseCase
└── presentation/
    ├── write/
    └── summary/
```

### 3.8 알림 모듈 (`notification`)

| 항목 | 내용 |
|:---|:---|
| **책임** | 푸시 알림, 인앱 알림 관리 |
| **핵심 기능** | 새 일자리 알림, 채팅 메시지 알림, 지원 결과 알림 |
| **외부 의존** | Firebase Cloud Messaging (FCM) |
| **시니어 특화** | 알림 빈도 제한(과다 알림 방지), 큰 텍스트 알림 스타일, 소리/진동 강조 |

```
notification/
├── data/
│   ├── repository/
│   └── datasource/          # FCM, Local DB
├── domain/
│   ├── model/               # Notification, NotificationSetting
│   └── usecase/             # SendPushUseCase, GetNotificationsUseCase
└── presentation/
    └── list/
```

## 4. 모듈 간 의존 관계

```
notification ←── chat ←── job ←── matching
                  ↑         ↑         ↑
                auth    location   profile
                  ↑         ↑         ↑
                  └─── core(공통) ───┘
                            ↑
                         review
```

### 의존 관계 원칙
1. **단방향 의존:** 순환 참조 금지, 상위 모듈만 하위 모듈 참조
2. **인터페이스 통신:** 모듈 간 통신은 도메인 레이어의 인터페이스를 통해 수행
3. **이벤트 기반:** 모듈 간 느슨한 결합이 필요한 경우 이벤트 버스 활용
4. **core 의존 최소화:** core 모듈은 비즈니스 로직 미포함, 순수 유틸리티만 제공

## 5. 기술 스택

| 영역 | 기술 | 선정 이유 |
|:---|:---|:---|
| **프레임워크** | Flutter (Dart) | 크로스플랫폼, 구형 기기 호환성, 빠른 개발 |
| **상태관리** | Riverpod | 모듈별 독립 상태 관리 용이 |
| **DI** | get_it + injectable | 모듈 간 의존성 주입 자동화 |
| **백엔드** | Firebase (Auth, Firestore, FCM, Storage) | 서버리스, 빠른 프로토타이핑 |
| **위치 서비스** | GeoFlutterFire + Kakao Map | Geo-query 최적화, 국내 지도 정확도 |
| **음성 인식** | Google STT API | 시니어 음성 입력 지원 |
| **CI/CD** | GitHub Actions + Fastlane | 모듈별 독립 빌드/배포 파이프라인 |

## 6. 개발 우선순위 및 단계별 로드맵

### Phase 1 — 기반 구축 (1~2주)
- [ ] core 모듈 세팅 (네트워크, 설정, 유틸)
- [ ] 공통 UI 컴포넌트 및 시니어 접근성 테마 구축
- [ ] Firebase 프로젝트 초기 설정
- [ ] CI/CD 파이프라인 구성

### Phase 2 — 핵심 기능 (3~6주)
- [ ] auth 모듈: 간편 로그인, 보호자 대리 가입
- [ ] profile 모듈: 간편 프로필 등록 (STT 포함)
- [ ] location 모듈: 동네 설정, GPS 연동
- [ ] job 모듈: 일자리 등록/조회/검색/지원

### Phase 3 — 매칭 및 커뮤니케이션 (7~10주)
- [ ] matching 모듈: 매칭 알고리즘, 역제안 기능
- [ ] chat 모듈: 실시간 1:1 채팅
- [ ] notification 모듈: 푸시 알림 연동

### Phase 4 — 신뢰 및 안정화 (11~12주)
- [ ] review 모듈: 상호 리뷰, 매너 온도
- [ ] 통합 테스트 및 QA
- [ ] 성능 최적화 (구형 기기 대응)
- [ ] 접근성 검수 (시니어 사용성 테스트)

## 7. 시니어 접근성 공통 가이드라인

모든 모듈이 준수해야 할 시니어 특화 UI/UX 원칙:

| 원칙 | 세부 기준 |
|:---|:---|
| **폰트 크기** | 최소 16sp, 제목 22sp 이상, OS 접근성 설정 연동 |
| **터치 영역** | 최소 48x48dp, 버튼 간 간격 12dp 이상 |
| **색상 대비** | WCAG AA 기준 4.5:1 이상 |
| **화면 뎁스** | 목적 달성까지 최대 3단계 이내 |
| **입력 방식** | 모든 텍스트 입력 필드에 STT 버튼 배치 |
| **피드백** | 동작 완료 시 시각 + 햅틱 피드백 동시 제공 |
| **네비게이션** | 하단 탭 바 고정, 최대 4개 탭 |
